# Noema — User Feedback Implementation Prompt

Update the existing **Noema** application based on the following real user-testing feedback.

Noema is a lightweight, open-source, local-first knowledge-management, research-journal and study application. It should combine:

* Notion-style organisation and page hierarchy
* Obsidian-style local Markdown files, vaults, linking and graph views
* A polished desktop-app experience
* Strong offline functionality
* Simple academic and exam-study workflows

Do not rebuild the application from scratch unless an architectural change is necessary. Preserve existing working functionality, branding and the Modrinth-inspired design direction.

The finished result must be functional. Do not add placeholder buttons, fake integrations or non-working interface elements.

---

# Main product direction

Noema should no longer feel primarily like a website.

It should feel like a real desktop productivity application that users can:

* Install on their computer
* Open from their taskbar
* Pin to their taskbar
* Launch from their desktop
* Keep running in their system tray
* Use without an internet connection
* Use without creating an account
* Use for university notes, mathematics, research, organisation and exam preparation

Maintain the web-based application architecture where practical, but package the application as a desktop app using **Tauri** or an equivalent lightweight, fully open-source desktop framework.

Avoid Electron unless there is a strong technical reason for using it.

Noema should continue to support:

* Local development in a browser
* Progressive Web App installation
* Desktop application installation
* Fully offline use
* Portable Markdown notes
* Optional self-hosting

The desktop version should be treated as the primary product experience.

---

# User profile and primary use cases

The feedback came from a university student who currently uses Notion.

Their main activities are:

* Learning lecture content
* Rewriting and paraphrasing lecture material
* Taking academic notes
* Working with advanced mathematics
* Preparing for exams
* Organising university tasks
* Searching for specific words or concepts
* Linking related pages
* Embedding lecture PDFs
* Using calendars
* Structuring information into pages and subpages

AI is not considered an essential feature.

Do not prioritise generative AI over core note-taking, navigation, search, organisation and offline reliability.

The primary value proposition should be:

> Noema helps students understand, organise, connect and revisit what they learn.

# Cross-device connectivity

Users want connectivity between their computer and phone.

Noema must remain local-first and open-source.

Implement a sync architecture that supports optional providers rather than forcing users onto a proprietary Noema cloud.

## Sync principles

* Sync is optional
* Offline editing remains available
* Local files remain the source of truth
* Users can export everything
* No Noema account is required
* Sync providers are replaceable
* Conflicts are never silently discarded
* End-to-end encryption should be supported for remote sync where practical

## Initial sync options

Support or design adapters for:

* Local network sync
* WebDAV
* Syncthing-compatible vault folders
* Git-based sync for advanced users
* User-selected cloud folders
* Optional self-hosted Noema sync server

The mobile experience may initially be:

* Installable PWA
* Responsive mobile interface
* Offline local vault
* Optional synchronisation through a supported adapter

Document limitations honestly.

## Conflict handling

When the same page changes on two devices:

* Keep both versions
* Show a conflict-resolution screen
* Highlight differences
* Allow the user to merge content
* Never overwrite a newer version silently

The updated Noema release is acceptable only when:

* Italic formatting works.
* Mathematics renders correctly and offline.
* The desktop app can be installed and launched normally.
* The application can be pinned to the taskbar.
* System-tray behaviour works.
* The application opens and edits notes without internet.
* Vaults are clearly explained.
* Users can create parent pages and subpages.
* Page hierarchy is stored portably.
* Search can find specific words and exact phrases.
* Search results clearly explain what was found.
* Users receive visible confirmation when important actions complete.
* Text and headings can be highlighted.
* PDFs can be embedded and viewed offline.
* Pages can be linked and embedded.
* Graph nodes can be coloured based on folder or parent structure.
* Calendar events can connect to pages.
* Tasks remain readable in Markdown.
* Exam-study workflows are usable.
* Slash commands are functional and searchable.
* Mobile use is supported.
* Cross-device sync is optional rather than mandatory.
* No account is required.
* No telemetry is introduced.
* All data can be exported.

The main goal of this iteration is not to make Noema larger for the sake of adding features.

The goal is to make Noema feel like a trustworthy, understandable and genuinely useful application for university students who need to learn, organise, connect and revisit complex information.
